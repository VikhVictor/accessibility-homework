# Карта намеренных дефектов

> Спойлер для преподавателя и сопровождающих проекта. В файле перечислены
> ожидаемые находки, поэтому студентам стоит открывать его после сдачи отчёта.

Репозиторий является учебным showcase. Дефекты доступности в обоих приложениях
намеренны и не должны исправляться как случайная часть рефакторинга.

- `:app` — исходная реализация на Android Views.
- `:app-compose` — те же экраны, поведение и проблемы TalkBack на Compose.
- `:data` — общий слой репозиториев и данных.

## Соответствие дефектов Views и Compose

| Учебный кейс | Android Views | Jetpack Compose |
| --- | --- | --- |
| Навигация без названий | У изображений «Назад» и «Корзина» нет `contentDescription` | `Image` для тех же действий используют `contentDescription = null` |
| Маленькие области нажатия | Иконки «Назад» и «Корзина» нажимаются в области 20dp | Автоматическое расширение области отключено, оба элемента остаются 20dp |
| Тихое обновление корзины | Счётчик — обычный текст без объявления изменения | У счётчика нет live-region semantics |
| Значимые изображения считаются декоративными | Изображения товара, пустой корзины, скидки, адреса, оплаты, выбора и стрелки не подписаны | Эквивалентные изображения намеренно не имеют описаний |
| Ненастоящие кнопки | Добавление в корзину и checkout реализованы как нажимаемые `TextView` | Используются нажимаемые `Text` без `Role.Button` |
| Нет структуры документа | Названия разделов — обычные `TextView` | У названий разделов нет heading semantics |
| Разорванные селекторы | Текст адреса/оплаты и checkbox являются отдельными целями фокуса | Текст и `Checkbox` также разделены, у checkbox нет доступного названия |
| Маленький переключатель доставки | Вариант меняет только неподписанная иконка 20dp | Вариант меняет только неподписанная иконка 20dp, строка не сообщает состояние выбора |
| Нет результата действия | Callback успешного/неуспешного checkout закомментирован | Подтверждение намеренно не даёт видимой или озвученной обратной связи |
| Навигация по каруселям | Вертикальные списки содержат горизонтальные | Используется такая же структура вложенных горизонтальных каруселей |

Инструментальные тесты Compose фиксируют репрезентативные дефекты. Падение такого
теста может означать, что кто-то случайно улучшил доступность. В этом проекте для
подобного изменения нужно отдельное решение в рамках учебного задания.

## Проверка

```shell
./gradlew :app:assembleDebug :app-compose:assembleDebug
./gradlew :app-compose:testDebugUnitTest
./gradlew :app-compose:connectedDebugAndroidTest
```
